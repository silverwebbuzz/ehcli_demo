What's removed (SaaS stripped out)

The following were SaaS-only features. Since this is a single-clinic private system for Dr. Feelgood's, all multi-tenancy, subscription tiers, and super admin features are completely removed. This makes the codebase simpler, faster to build, and easier to maintain.

Multi-tenancy (clinic_id everywhere)One clinic — no need to isolate data by tenantRemoved
Subdomain routing (clinic1.myclinic.in)Single URL: app.drfeelgoods.inRemoved
Subscription tiers (Basic/Pro/Enterprise)No monetisation layer neededRemoved
Super Admin panelNo platform-level management neededRemoved
Tenant onboarding wizardSystem is pre-installed, no self-signupRemoved
Razorpay subscription billingClinic is the customer, not the SaaS buyerRemoved
Plan enforcement middlewareAll features unlocked for this single clinicRemoved
tenants tableNo more tenant isolation — global config file insteadRemoved

MySQL 8 — Complete CREATE TABLE statements
No clinic_id · No tenants table · Homeopathy fields included
-- DATABASE
CREATE DATABASE IF NOT EXISTS drfeelgoods
  CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE drfeelgoods;
-- TABLE: users (staff + patient portal)
CREATE TABLE users (
  id            BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  name          VARCHAR(100) NOT NULL,
  email         VARCHAR(150) NOT NULL,
  phone         VARCHAR(15),
  password_hash VARCHAR(255) NOT NULL,
  role          ENUM('admin','receptionist','assistant','patient')
                NOT NULL DEFAULT 'receptionist',
  is_active     TINYINT(1) DEFAULT 1,
  remember_token VARCHAR(100),
  last_login_at  TIMESTAMP NULL,
  created_at    TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uq_email (email)
) ENGINE=InnoDB;
-- TABLE: patients (homeopathy-adapted)
CREATE TABLE patients (
  id              BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  user_id         BIGINT UNSIGNED NULL,  -- patient portal login
  uhid            VARCHAR(20) NOT NULL,  -- DFG-00001
  name            VARCHAR(100) NOT NULL,
  dob             DATE,
  gender          ENUM('M','F','Other'),
  phone           VARCHAR(15) NOT NULL,
  email           VARCHAR(150),
  address         TEXT,
  blood_group     ENUM('A+','A-','B+','B-','O+','O-','AB+','AB-'),
  veg_type        ENUM('veg','nonveg','vegan','eggetarian') DEFAULT 'veg',
  allergies       TEXT,
  chronic_conditions TEXT,
  -- Homeopathy-specific fields
  constitution_type ENUM('psoric','sycotic','syphilitic',
                         'tubercular','mixed'),
  thermal_sensitivity ENUM('hot_patient','cold_patient','chilly'),
  food_cravings   TEXT,
  food_aversions  TEXT,
  mental_state_notes TEXT,
  -- Media
  photo_path      VARCHAR(255),
  qr_token        VARCHAR(64) NOT NULL,
  is_active       TINYINT(1) DEFAULT 1,
  created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  UNIQUE KEY uq_uhid (uhid),
  UNIQUE KEY uq_qr (qr_token),
  KEY idx_phone (phone),
  KEY idx_name (name),
  FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL
) ENGINE=InnoDB;
-- TABLE: remedies (homeopathic remedy master)
CREATE TABLE remedies (
  id                  BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  name                VARCHAR(100) NOT NULL,  -- Arsenicum Album
  abbreviation        VARCHAR(20),             -- Ars.alb
  source              ENUM('plant','mineral','animal','nosode','sarcode'),
  key_indications     TEXT,
  antidotes           JSON,  -- ["Camphora","Nux Vomica"]
  complementaries     JSON,  -- compatible follow-up remedies
  dietary_restrictions TEXT, -- "Avoid: coffee, mint, camphor, raw onion"
  PRIMARY KEY (id),
  FULLTEXT idx_search (name, abbreviation, key_indications)
) ENGINE=InnoDB;
-- TABLE: visits (extended case-taking)
CREATE TABLE visits (
  id                 BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  appointment_id     BIGINT UNSIGNED NULL,
  patient_id         BIGINT UNSIGNED NOT NULL,
  chief_complaint    TEXT,
  -- Homeopathy case-taking columns
  mental_generals    TEXT,  -- anxiety, fears, irritability, grief...
  physical_generals  TEXT,  -- sleep, appetite, thirst, sweat, thermals
  peculiar_symptoms  TEXT,  -- strange/rare/peculiar symptoms (key in homeo)
  modalities         TEXT,  -- better/worse from heat, cold, motion, rest
  miasmatic_analysis TEXT,  -- psoric/sycotic/syphilitic presentation
  -- Clinical
  diagnosis          TEXT,
  clinical_notes     LONGTEXT,
  condition_score    TINYINT UNSIGNED,  -- 1-10 improvement rating
  follow_up_date     DATE,
  visited_at         TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  KEY idx_patient_visits (patient_id, visited_at),
  FOREIGN KEY (patient_id) REFERENCES patients(id)
) ENGINE=InnoDB;
-- TABLE: prescriptions (homeopathic)
CREATE TABLE prescriptions (
  id           BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  visit_id     BIGINT UNSIGNED NOT NULL,
  patient_id   BIGINT UNSIGNED NOT NULL,
  remedy_id    BIGINT UNSIGNED NOT NULL,
  potency      ENUM('6C','12C','30C','200C','1M','10M','CM','Q','LM1','LM2'),
  form         ENUM('pellets','liquid','powder','mother_tincture','trituration'),
  dosage       VARCHAR(60),  -- "4 pills" / "10 drops in water"
  frequency    ENUM('OD','BD','TDS','weekly','monthly','SOS'),
  duration_days SMALLINT,
  instructions  TEXT,  -- "Dissolve under tongue", "30 min before meals"
  PRIMARY KEY (id),
  KEY idx_visit (visit_id),
  FOREIGN KEY (visit_id)   REFERENCES visits(id),
  FOREIGN KEY (remedy_id)  REFERENCES remedies(id),
  FOREIGN KEY (patient_id) REFERENCES patients(id)
) ENGINE=InnoDB;
-- TABLE: vitals (with thyroid + skin score)
CREATE TABLE vitals (
  id          BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  visit_id    BIGINT UNSIGNED NOT NULL,
  patient_id  BIGINT UNSIGNED NOT NULL,
  bp_systolic  SMALLINT UNSIGNED,
  bp_diastolic SMALLINT UNSIGNED,
  blood_sugar  DECIMAL(5,1),
  weight_kg    DECIMAL(5,2),
  height_cm    DECIMAL(5,1),
  bmi          DECIMAL(4,1) GENERATED ALWAYS AS
               (weight_kg / ((height_cm/100) * (height_cm/100))) STORED,
  temperature  DECIMAL(4,1),
  spo2         TINYINT UNSIGNED,
  pulse_rate   SMALLINT UNSIGNED,
  -- Thyroid panel (key condition at Dr. Feelgood's)
  tsh          DECIMAL(6,3),  -- normal 0.4-4.0 mIU/L
  t3           DECIMAL(6,2),
  t4           DECIMAL(6,2),
  -- Skin improvement score (psoriasis, eczema tracking)
  skin_score   TINYINT UNSIGNED,  -- 1 (worst) to 10 (cleared)
  recorded_at  TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  PRIMARY KEY (id),
  KEY idx_patient_vitals (patient_id, recorded_at),
  FOREIGN KEY (visit_id)   REFERENCES visits(id),
  FOREIGN KEY (patient_id) REFERENCES patients(id)
) ENGINE=InnoDB;
-- TABLE: settings (replaces tenants table)
CREATE TABLE settings (
  `key`   VARCHAR(60) NOT NULL,
  `value` TEXT,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`key`)
) ENGINE=InnoDB;

app.drfeelgoods.in — System Architecture
PHP 8.2 MVC · MySQL 8 · No multi-tenancy · Single server deployment
1
Presentation — what users see
Staff App
app.drfeelgoods.in — Tailwind CSS + Alpine.js. Role-filtered sidebar. All clinic modules.
Patient Portal
app.drfeelgoods.in/portal — OTP login. View history, Rx, invoices, book appointment.
QR Scanner
Mobile camera API. Scan QR → /qr/{token} → instant patient profile load.
Print views
Prescription PDF, Invoice PDF, QR patient card PDF — all server-rendered, branded.
2
Application — PHP 8.2 MVC
Router
Custom PHP router. /api/* → JSON responses. /* → HTML views. Named routes.
Auth Middleware
JWT in HttpOnly cookie. Session fallback. OTP for patient portal.
RBAC
4 roles: admin, receptionist, assistant, patient. Permission config array checked per route.
Controllers
PatientController, AppointmentController, VisitController, PrescriptionController, BillingController…
Services
NotificationService, PdfService, QrService, RemedyService, BillingService
CSRF + Validation
CSRF token on every form. Server-side validation. Sanitize all inputs.
3
Data — MySQL 8 + Redis
MySQL 8
18 tables. InnoDB. UTF8MB4. PDO prepared statements. No ORM bloat.
Redis (optional)
Session store. Dashboard stats cache (5 min TTL). Remedy search cache.
File Storage
/storage/patients/{id}/ — photos, lab PDFs, generated prescriptions. Served via PHP (auth check).
Settings table
Key-value store for all clinic config. Replaces .env for editable values. Admin UI to change.
4
Background workers — PHP cron
Notification worker
Cron every 5 min. Reads notifications table (queued). Sends WhatsApp/SMS. Marks sent/failed.
Reminder scheduler
Cron 8AM daily. Finds tomorrow's appointments → queues WhatsApp reminders.
Follow-up alerts
Cron daily. Patients with follow_up_date = tomorrow → WhatsApp reminder.
5
External integrations
WhatsApp Business API
Meta Cloud API (free). Templates: appointment_reminder, prescription_ready, lab_report, diet_plan, follow_up_reminder.
Razorpay
UPI QR generation at billing desk. Webhook → marks invoice paid. No subscription module.
mPDF
Prescription PDF (Dr. Feelgood's letterhead), GST Invoice PDF, QR patient card PDF, diet plan PDF.
endroid/qr-code
Generates QR PNG for patient card. QR encodes app.drfeelgoods.in/qr/{token}
SMS (2Factor.in)
OTP for patient portal login. Fallback when WhatsApp fails.
6
Deployment — single server
Server
Ubuntu 22.04 VPS (2GB RAM, 2 vCPU sufficient). Hostinger / DigitalOcean / Hetzner.
Nginx + PHP-FPM
Nginx serves static files. PHP-FPM handles app. app.drfeelgoods.in → /var/www/drfeelgoods
SSL
Let's Encrypt cert for app.drfeelgoods.in. Auto-renew via certbot.
Backups
Daily mysqldump to /backups + sync to Google Drive or S3. 30-day retention.

Appointments + Queue
Slot booking, walk-in, today's queue, reminders

▼

Slot availability engine
Based on clinic hours in settings table. 30-min slots. Morning / Evening / Sunday sessions. Available slots = total slots − booked appointments for that date.
PHPMySQL

Booking + walk-in interface
Receptionist: select patient → date → slot → confirm. Walk-in: patient search (or quick register) → auto-assign next token number for current session.

Today's queue display
Real-time token list (auto-refresh every 30s via AJAX). Current patient highlighted. Status buttons: Mark arrived → In consultation → Completed. Doctor sees next patient name.

WhatsApp appointment reminder
Cron 8AM daily → find tomorrow's appointments → queue WhatsApp messages. Template: "Dear {name}, reminder for your appointment at Dr. Feelgood's tomorrow at {time}. Reply CANCEL to cancel."
CronWhatsApp API

Clinical Visit + Homeopathic Prescription

▼

Visit screen with homeo case-taking tabs
Doctor opens patient from queue. Tabs: Chief Complaint | Case Taking (mental generals, physical generals, peculiar symptoms, modalities, miasmatic analysis) | Vitals | Prescription | Notes. Auto-saves visit on close.

Vitals entry + condition score
Quick form: BP, blood sugar, weight, temp, SpO2. Thyroid: TSH/T3/T4. Skin score: 1–10 slider with label (1=severe, 10=cleared). BMI auto-calculates. Charts load last 10 readings instantly.

Potency + form + dosage selector
